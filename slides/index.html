Conclusions

Pros

  • Vs. Backbone: Much more automatic management of view lifecycle (primarily via regions)
  • Template/Rendering pipeline
  • Patterns for more declarative code: UI, ModelEvents, CollectionEvents, etc.
  • Templates have a minimum of logic in them “separation of concerns”
  • Debuggable

Cons

  • You must write more LOC
  • Many files (if you modularize views)
  • Every view instance must be associated with a single DOM element
  • Can lead to non-semantic DOM if you don’t take care
  • Many ways to handle cross-view communication .. no clear ‘best’ way
  • app.vent
  • layouts
  • model/collection events
